# 프론트엔드 규칙

## 기술과 상태 관리

- Next.js 15 이상, App Router, React 18 이상, TypeScript 5를 사용한다.
- Tailwind CSS 4와 Axios를 사용한다.
- Server Component를 기본값으로 유지한다.
- 브라우저 API와 상태가 필요한 최소 컴포넌트에만 `"use client"`를 선언한다.
- React 기본 상태와 Effect로 클라이언트 상태를 관리한다.
- 컴포넌트 해제 후 상태 갱신과 오래된 요청의 응답 반영을 방지한다.
- 컴포넌트 Props와 API 데이터 타입을 명시적으로 정의한다.
- `any` 사용을 피하고 외부 응답을 신뢰하기 전에 검증한다.

## 페이지 동작

### `/`

- 서버에서 한국 표준시 기준 오늘 날짜를 계산한다.
- `/todos?due_date=YYYY-MM-DD&status=all`로 리다이렉트한다.

### `/todos`

- Todo 목록과 날짜 및 완료 상태 필터를 제공한다.
- 완료 상태 전환과 삭제를 목록에서 처리한다.
- 삭제 전 브라우저 기본 확인창으로 사용자 확인을 받는다.
- 필터는 `/todos?due_date=YYYY-MM-DD&status=all|active|completed` 형태의 URL 쿼리로 관리한다.
- 잘못되거나 누락된 날짜는 한국 기준 오늘 날짜로 보정한다.
- 잘못되거나 누락된 상태는 `all`로 보정한다.

### `/todos/new`

- 제목만 입력받는다.
- `due_date`는 진입한 목록의 URL 날짜를 사용한다.
- 유효한 날짜가 없으면 한국 기준 오늘 날짜를 사용한다.
- 생성 성공 후 생성된 Todo 날짜의 `status=all` 목록으로 이동한다.

### `/todos/[todoId]`

- 기존 Todo를 조회하고 제목만 수정한다.
- 날짜와 완료 상태는 수정하지 않는다.
- 수정 성공 후 Todo 날짜의 `status=all` 목록으로 이동한다.
- 존재하지 않는 Todo는 적절한 404 화면 또는 오류 상태로 처리한다.

## 로딩과 오류

- 최초 페이지 로딩과 페이지 이동에는 `loading.tsx`를 사용한다.
- 페이지 자체를 불러오지 못하면 `error.tsx`를 사용한다.
- CRUD 요청 중에는 요청을 발생시킨 폼이나 Todo 항목만 비활성화한다.
- 중복 제출과 중복 상태 변경을 방지한다.
- CRUD 오류는 관련 폼이나 목록 근처에 표시한다.
- 요청 실패 시 사용자가 입력한 제목을 보존한다.
- 내부 예외, 스택 트레이스, 민감한 백엔드 정보를 노출하지 않는다.

## UI와 접근성

- 기존 보라색 계열 디자인을 유지한다.
- 모바일과 데스크톱에서 사용할 수 있는 반응형 레이아웃을 제공한다.
- 모든 입력에는 연결된 `label`을 제공한다.
- 모든 동작은 키보드로 실행할 수 있어야 한다.
- 명확한 포커스 표시를 유지한다.
- 완료, 선택, 오류 상태를 색상만으로 전달하지 않는다.
- 로딩 및 오류 메시지에 필요한 접근성 속성을 적용한다.

## 환경과 스타일

```dotenv
# frontend/.env.local
BACKEND_API_URL=http://localhost:8000
```

- 실제 `.env.local`은 커밋하지 않고 `.env.example`을 제공한다.
- `NEXT_PUBLIC_` 접두사가 붙은 백엔드 URL을 만들지 않는다.
- 기본 로컬 포트는 `3000`이다.
- TypeScript strict 모드를 유지한다.
- ESLint 오류를 무시하거나 광범위하게 비활성화하지 않는다.
- Prettier를 단일 포맷 기준으로 사용한다.
- 재사용 가치가 있는 API 타입과 도우미만 분리한다.
